# TensuGo 新功能设计 —— BRG 围棋研究文档（期评）系统

你现在是一名资深产品架构师、Go GUI 开发者、文档格式设计专家。

## 项目背景

TensuGo 已经完成了传统围棋 GUI 的主体功能，包括：

* SGF / GIB 棋谱读取
* KataGo 分析
* 候选点
* PV
* 自动分析
* 分支树
* 摆棋
* 保存变化
* 写棋评（初版）
* 导出 PDF

目前整体功能已经达到原 TensuGo UI 大约 70% 的能力。

这个文档是关于一个任何其它围棋软件都没有的能力：

> **BRG（BlackRice Go Research）围棋研究文档系统。**

---

# 产品定位

BRG 不是棋谱，不是 SGF，不是 GIB。而是：
**围棋研究文档（Research Document）。**

一个 BRG 文档应该能够完整保存：

* 棋谱
* 研究过程
* AI 分析
* 多个变化图
* 图文讲解
* 作者观点
* 排版结构

最后生成可以长期保存、分享、传播的专业期评。

---

# 文档格式

采用：

```
.brg
```

底层采用 JSON。

SGF、GIB 只是 BRG 的输入来源。

BRG 才是最终研究成果。

请设计：

## JSON Schema

至少包含：

* version
* title
* author
* createdAt
* updatedAt

棋谱信息：

* sourceFormat（sgf / gib）
* originalFile
* players
* result
* rules
* komi

文档：

* sections
* blocks

---

# Block 设计

BRG 采用 Block 结构。

支持：

```
Heading
Paragraph
Board
Variation
AIAnalysis
CandidateMoves
Quote
Image
Conclusion
```

重点设计：

## Board Block

保存：

* 当前局面
* 当前手数
* 是否显示坐标
* 标记
* 箭头
* 三角
* 数字
* 标签

支持：

点击即可恢复桌面棋盘状态。

---

## Variation Block

保存：

* 起始局面
* 多条变化
* 每条变化标题
* 每条变化说明

要求：

以后网站可以直接播放变化。

---

## AIAnalysis Block

保存：

* KataGo
* visits
* winrate
* scoreLead
* PV
* candidateMoves

要求：

研究文档打开以后，无需重新计算即可查看历史分析。

---

## Paragraph Block

支持 Markdown。

支持：

* 标题
* 列表
* 粗体
* 引用

即可。

不要复杂排版。

# 桌面端工作流

研究流程应该是：

打开 SGF 或 GIB

↓

浏览棋谱

↓

分析

↓

找到关键局面

↓

点击：

```
插入局面
```

生成一个 Board Block。

然后：

```
写讲解
```

继续：

```
插入 AI 分析
```

生成 AIAnalysis Block。

继续：

```
保存变化
```

生成 Variation Block。

不断重复。

最终形成完整期评。

整个文档就是：

主要实现PDF导出后看到棋局图和对应文字评论，符合大众多年围棋天地杂志的阅读习惯。

# PDF 导出

BRG 最终成果：

导出 PDF。

PDF 不是截图。

而是一份正式围棋研究报告。

要求：

封面：

标题

作者

日期

正文：

图文混排。

棋盘高清。

变化图自动排版。

AI 数据保持整洁。

页脚：

页码:

最后：

总结。

要求：

适合：

微信群，公众号，邮件 传播长期保存。

建议采用：

HTML 渲染布局 + Headless Chromium 导出 PDF。

保证棋盘全部采用 SVG，打印清晰。
