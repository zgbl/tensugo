# TensuGo Release v0.1 功能范围

本文件记录 TensuGo Release v0.1 线的功能取舍和实现约束。当前安装包版本使用 `0.1.x` patch/build 号；每次打包安装都必须自动让 patch 版本 +1。

## 0. 截图对应的主界面区域

| 功能 | 说明 | 决策 |
| --- | --- | --- |
| 窗口标题栏 | 显示当前分析摘要、引擎、棋手、文件名 | MUST |
| 顶部菜单栏 | 文件、显示、棋局、分析、编辑、同步、帮助、设置、跑谱贡献、当前引擎 | MUST |
| 顶部图标工具栏 | 新建、打开、保存、闪电分析、天书、禁入/允许、刷新/同步、撤销等快捷按钮 | MUST |
| 顶部参数栏 | 新对局、暂停、贴目、激进度、广度、选点黑白、规则、死活、参数、棋盘、存档 | SOON |
| 左侧信息面板 | 规则、黑白棋子、吻合度、贴目、提子、胜率条、目差/胜率图、当前局面文字摘要 | MUST |
| 中央主棋盘 | 棋盘、棋子、候选点气泡、候选点序号、胜率/计算量/目差显示、最后一手显示 | MUST |
| 右侧竖向变化/手数条 | 棋局主线/分支手数导航，当前手高亮 | MUST |
| 右侧候选点表 | 序号、胜率、计算量、占比、目数 | MUST |
| 右下小棋盘 | 全局缩略棋盘/变化图预览 | SOON |
| 底部工具栏 | 同步、AI分析、自动分析/暂停分析/继续分析、结束分析、纯网络、设为主分支、清空棋盘、删除、跳转、形势判断、手数、坐标、自动播放 | MUST |

启动约束：

- App 启动后显示空棋盘，不预载示例棋谱。

## 21.1 胜率变化图 / 主线进度控制

| 功能 | 说明 | 决策 |
| --- | --- | --- |
| 整盘胜率曲线 | 显示主线各分析点胜率变化 | MUST |

| 点击图表跳转 | 点击横向位置后主棋盘跳到对应主线手数 | MUST |
| 分析点缓存 | KataGo 返回一手分析后记录该手胜率/目差/计算量 | MUST |
| 批量补全整盘曲线 | 对整盘主线逐手分析，填满曲线 | MUST |
| 自动分析设置 | 设置开始/结束手数、每手秒数、每手 PO、黑白过滤 | MUST |
| 吻合率统计 | 统计实战手是否在候选点中，以及是否等于第一候选 | MUST |

实现约束：

- 胜率变化图放在左侧信息栏，不占用右侧 PV 小棋盘空间。
- 自动分析按“实战手之前的局面”逐手调用 KataGo，用候选点和下一手实战落子计算吻合率。
- 实时 AI 分析不能阻塞用户落子、跳转、摆变化等主操作；用户操作发生时应取消/作废旧分析，短暂空闲后再排队新分析。
- 自动分析的“每手时间(秒)”是每个被分析手的最短墙钟耗时；引擎更快返回时需要等待剩余时间，避免一秒跳多手。
- 自动分析运行时，底部按钮显示 `暂停分析`；暂停后保留下一手断点，按钮显示 `继续分析`，点击后从断点继续。
- `结束分析` 与暂停分离：结束后清除继续断点，并打开天书报告显示总统计信息。
- 天书报告按旧 UI 信息架构搭建：吻合度、吻合率、第一选命中率、胜率变化图、目数损失统计、胜率损失统计、底部摘要与统计条件。
- 当前吻合度先按候选名次加权估算：首选为满分，其它候选按名次递减，未进候选为 0；后续可按正式算法替换。
- 自动分析每一步必须显示该局面的候选点气泡/候选表，分支树高亮跟随当前分析位置，胜率图在每步结果返回后追加/更新分析点。
- 自动分析应使用短批次引擎请求反复刷新当前局面，避免单次长阻塞导致 UI 像死机，并让停止按钮在批次间生效。
- 自动分析如果某一步没有返回候选点，应停止并显示失败原因，不能继续只移动棋谱手数。
