实现 TensuGo 的“鹰眼报告 / AI吻合度统计”。

背景：
参考 LizzieYZY / KataGo UI 的超级鹰眼逻辑，先还原计算方法，以便兼容大家多年习惯的讨论术语和数据。
TensuGo 需要统计每一手与 AI 推荐候选点的吻合情况，并在自动分析摘要和 PDF/HTML 导出中显示。

核心概念：

1. 吻合度 matchScore
对每一手，如果实战落点在 KataGo candidates 中，则：

matchScore = actualCandidate.visits / maxCandidate.visits

其中：
- actualCandidate 是实战落点对应的候选点
- maxCandidate.visits 是当前局面所有候选点中 visits 最大值
- 如果找不到实战落点，matchScore = 0

最终平均吻合度：

averageMatchScore = sum(matchScore) / analyzedMoveCount * 100

显示为百分比。

2. 吻合率 matchRate
这是命中比例，不是平均分。

判断某一手是否命中 AI 候选，需要同时满足：

- actualCandidateIndex >= 0
- actualCandidateIndex < matchSettings.bestNums
- matchScore * 100 >= matchSettings.percentVisits

如果满足，isMatch = true。

最终：

matchRate = matchedMoveCount / analyzedMoveCount * 100

3. Top Move 命中率 topMoveRate

topMoveRate = topMoveMatchCount / analyzedMoveCount * 100

其中 topMoveMatchCount 是 actualCandidateIndex === 0 的手数。

4. Candidate 命中率 candidateRate

candidateRate = candidateMatchCount / analyzedMoveCount * 100

其中 candidateMatchCount 是 actualCandidateIndex >= 0 的手数，不受 bestNums 和 percentVisits 限制。

5. 胜率损失 winrateLoss

如果实战点在候选中：

winrateLoss = max(0, best.winrate - actualCandidate.winrate)

否则胜率损失为未知，不参与平均胜率损失和胜率损失分桶统计。

最终：

averageWinrateLoss = totalWinrateLoss / knownWinrateLossCount

6. 目差损失 scoreLoss

如果能找到 actualCandidate，则根据当前行棋方计算 scoreLoss。

保留现有 calculateScoreLoss(...) 逻辑。

最终：

averageScoreLoss = totalScoreLoss / knownScoreLossCount

不要在 unknown/null 情况下强行计入 0。

实现要求：

1. 把现有 summary 统计逻辑统一到一个纯函数中，例如：

calculateAutoAnalysisSummary(results, gameTree, matchSettings)

不要把统计逻辑散落在 React UI 里。

2. 每分析完一手，可以增量更新 summary，但最终算法必须和纯函数一致。

3. 命名避免拼写错误：
不要用 accracy / precents。
统一使用：
accuracy / percentage / matchScore / matchRate。

4. UI 显示中文：

- 平均吻合度
- 吻合率
- 最佳一选命中率
- 候选命中率
- 平均胜率损失
- 平均目差损失
- 最大失误

5. PDF/HTML 鹰眼报告中显示：

鹰眼报告：
- 分析手数
- 平均吻合度
- 吻合率
- 最佳一选命中率
- 候选命中率
- 平均胜率损失
- 平均目差损失
- 最大失误

6. 不要把完整 KataGo raw analysis 保存进 .tsg。
只保存 summary 和必要 detail。
不要保存 ownership/policy/raw response。

7. 添加简单单元测试：

测试场景：
best visits = 1000

实际下在：
- rank 1, visits 1000 => matchScore 1.0, isTopMove true, isMatch true
- rank 2, visits 600 => matchScore 0.6, isTopMove false, isCandidate true
- rank 5, visits 100 => 如果 bestNums=3，则 isMatch false
- 未找到候选 => matchScore 0, isCandidate false, isMatch false

验收：
- 平均吻合度等于 matchScore 平均值 * 100
- 吻合率等于 isMatch true 的比例
- Top Move 命中率等于 rank 1 比例
- Candidate 命中率等于找到候选的比例

公式：
========================================
Match Score (每手吻合度)
========================================

如果实战点在 AI 候选中：

matchScore = actualCandidate.visits / bestCandidate.visits

其中：

bestCandidate.visits
    当前局面 visits 最大的候选点

actualCandidate.visits
    实战落点对应候选点的 visits

如果实战点不在候选中：

matchScore = 0

----------------------------------------

平均吻合度：

averageMatchScore =
    sum(matchScore)
    -----------------------
      analyzedMoveCount

显示时乘以 100，作为百分比。

========================================
Match Rate (吻合率)
========================================

一手只有满足下面两个条件才算命中：

1.
actualCandidate.rank <= bestNums

2.
matchScore >= percentVisits / 100

例如：

bestNums = 3

percentVisits = 20

则：

Rank 2
Visits Ratio = 35%

=> 命中

----------------------------------------

matchRate =
    matchedMoveCount
    -------------------------
      analyzedMoveCount

显示时乘以 100。

========================================
Top Move Rate
========================================

如果：

actualCandidate.rank == 1

则：

topMoveMatchCount ++

最后：

topMoveRate =
    topMoveMatchCount
    --------------------------
      analyzedMoveCount

========================================
Candidate Rate
========================================

如果：

actualCandidate 存在

即：

actualCandidateIndex >= 0

则：

candidateMatchCount ++

最后：

candidateRate =
    candidateMatchCount
    --------------------------
      analyzedMoveCount

========================================
Winrate Loss
========================================

如果实战点在候选中：

winrateLoss =
    best.winrate
    -
    actualCandidate.winrate

如果结果小于 0：

winrateLoss = 0

如果没有找到实战点：

winrateLoss =
    100
    -
    best.winrate

----------------------------------------

averageWinrateLoss =
    totalWinrateLoss
    -------------------------
      analyzedMoveCount
