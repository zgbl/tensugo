昨天因为完成了拟人引擎的配置，今天对出题的模式有如下新设计：

1> 题目我是打算分两种题。
    1.1 A型题，是不显示候选点，让用户像下棋一样选择自己的下一手。题目里保存了12个AI候选点。如果答题人命中1选，得10分。命中2-5选，大致得9.5,9.0,8.5,8.0分，其他候选依次降低到5分。如果不在12候选之内，不得分。
    1.2，B腥题，这是我今天要重点设计的，会显示候选点的题目类型。这种就像普通的考试一样，显示A，B，C，D，E 候选点答案。下面要重点说。

2> 对于B型题。由于现在有了拟人引擎，现在有了批量出题的可能性。
    2.1> 今天第一步，先实现出一道题
    2.2>题有5个候选点：1.手是原棋谱的点。2. 强AI 1选点。3，强AI 第3选点，4-5. 拟人AI前三选点中的 2个。
    2.3> 拟人AI实力的强弱设定：系统解析棋谱Meta Data 棋手段位，比如如果这个棋手是野狐或者弈城7D，则这个题为7D水平的题。拟人段位本应该也是7D，但是由于经过实测，你让引擎实力过于超出标称段位。所以我暂时以其标称段位-5段，（就是用2D作为7D级别题的拟人引擎）这个设定不要写死，要在设定里可以设置。
3> 出题引擎的设置：
    要在UI设置里加一个tab, 出题。下面参数需要设置：
    3.1：B型题候选点的数量（默认 5）
    3.2：拟人引擎实力 offset： 默认： -5  ， -5 表示比题目级别减5段/级
    3.3：拟人段位负责几个候选 ，默认： 2


4> 出题模式要能预览，预览出好的题是什么样子，就是进入做题模式，显示答题人会看到的这道题的样子。
    我点”预览题目“，棋盘上应该显示5个候选点标记 A, B, C, D, E, 几个点的位置。AI分析信息哟啊保留显示。

5> 如果三个候选点来源的选点重复了。优先级依次为: 强AI 1选 > 实战实际选点> 强AI 次选 > 拟人引擎选点，优先级低的点要向下选本序列下一个候选点递补上来做候选点。 

6> 题目候选点的打分：
  6.1> 正确一选永远是10分。
  6.2> 强 AI 次选 （2-5选）按顺序给9.5,9.0,8.5,8.0分。
  6.3> 实战选点（通常是没下好，损失胜率了）和拟人引擎选点，靠强AI重新评分。
  6.4> 把5个选点重新喂给强AI打分，除了正确选点外的四个选点，只打分，不改选点。但是10分正确选点，如果后来AI再次评分的事后，visit 数字超过远分析visit数的2倍，并且更改了选点，那就按新的选点（如果和后面几个不冲突）升级成正确选点。如果冲突，就剩四个答案，等人工处理。
  强AI评分，胜率损损失每多0.2%，扣一分。到5分，然后每多0.5%扣一分，扣光为止。

7> 做题：
做题当然不能再显示AI分析信息。
棋盘显示 A-E 选点。右边要给我答题区。让我填哪个是正确的。或者在棋盘上点要回答哪个点。然后点下一题。全做完了，显示得分。
做题模式下，禁止开AI分析，disable

记得我们说了有A型题和B型题吧？就是两种模式。做题模式应该能选A型还是B型模式下做题。
这个12个选择点的，是给A型题的，A型题做题不应该显示任何提示，由用户自由在棋盘上点，然后提交。
B型呢，要显示候选点，但是只显示5个。
所以现在UI，哪个都不符合。你要再改。

1> 字母改成深蓝色，字体不要太大。不要加粗。
2> 不要加圈，更不要填充，否则容易让人有每个点都是已经落在棋盘上的子的错觉，十分妨碍视觉判断。
